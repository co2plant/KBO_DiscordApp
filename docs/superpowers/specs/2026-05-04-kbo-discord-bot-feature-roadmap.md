# KBO Discord Bot Feature Roadmap

> 참고용 이전 로드맵입니다. 현재 기준 문서는 `docs/ROADMAP.md`입니다.

작성일: 2026-05-04

## 목적

KBO 팬이 디스코드 안에서 경기 일정, 실시간 상황, 순위, 응원팀 알림을 빠르게 확인할 수 있는 봇을 만든다. 핵심 가치는 "지금 경기 어떻게 됐어?"라는 질문에 가장 짧은 흐름으로 답하는 것이다.

## 진행 원칙

- 기능은 한 번에 모두 구현하지 않고, 사용자 가치가 큰 순서대로 작은 단위로 진행한다.
- 각 단계는 목표, 범위, 제외 범위, 검증 조건을 먼저 정리한 뒤 구현한다.
- 서버별 설정과 데이터 안정성은 초반부터 고려하되, 과한 기능은 뒤 단계로 미룬다.
- KBO 데이터 출처가 불안정할 수 있으므로 캐시, 실패 메시지, 재시도 전략을 함께 설계한다.

## Phase 1: MVP 정보 조회

목표: 봇을 초대하면 바로 쓸 수 있는 기본 조회 기능을 제공한다.

기능:
- `/오늘경기`: 오늘 KBO 경기 일정, 구장, 시작 시간, 경기 상태 표시
- `/스코어`: 현재 진행 중인 경기의 실시간 점수 표시
- `/순위`: 팀 순위, 승패, 승률, 게임차 표시
- `/팀`: 특정 팀의 오늘 경기, 최근 상태, 순위 요약 표시
- 우천취소, 연기, 경기 지연 상태 표시

완료 기준:
- 각 명령어가 정상 데이터, 경기 없음, 데이터 수집 실패 상황을 구분해서 응답한다.
- 모든 시간 표시는 한국 시간 기준으로 일관되게 표시한다.
- Discord 메시지는 모바일에서도 읽기 쉬운 짧은 카드 또는 embed 형태로 제공한다.

## Phase 2: 서버별 설정과 기본 알림

목표: 서버마다 원하는 팀과 알림 채널을 설정할 수 있게 한다.

기능:
- 서버별 응원팀 설정
- 경기 알림 채널 설정
- 경기 시작 알림
- 경기 종료 알림
- 취소, 연기, 지연 알림
- 팀별 알림 필터

완료 기준:
- 서버별 설정이 영구 저장된다.
- 설정이 없는 서버에서도 기본 조회 명령은 문제없이 동작한다.
- 알림 중복 발송을 방지한다.

## Phase 3: 라인업과 주요 경기 이벤트

목표: 경기 전후와 경기 중에 팬들이 가장 반응하는 순간을 자동으로 알려준다.

기능:
- 선발투수 발표 알림
- 선발 라인업 발표 알림
- 득점 이벤트 알림
- 홈런 이벤트 알림
- 역전, 동점, 끝내기 상황 강조 알림
- 경기 종료 후 자동 요약

완료 기준:
- 이벤트 알림은 서버 설정과 팀 필터를 따른다.
- 같은 이벤트를 반복 발송하지 않는다.
- 경기 종료 요약에는 최종 스코어와 핵심 장면이 포함된다.

## Phase 4: 팀/선수 심화 정보

목표: 경기 외에도 팬들이 자주 찾는 팀과 선수 정보를 제공한다.

기능:
- 선수 검색: 타율, 홈런, ERA, WHIP 등 주요 지표
- 팀 최근 5경기 흐름
- 팀 간 상대 전적
- 선발 로테이션 예상
- 1군 등록/말소 정보
- 부상, 이적, 외국인 선수 관련 뉴스 요약

완료 기준:
- 선수와 팀 이름의 한글 검색이 자연스럽게 동작한다.
- 데이터가 없거나 불확실한 항목은 추정처럼 보이지 않게 표시한다.
- 뉴스 요약은 출처와 시간을 함께 제공한다.

## Phase 5: 커뮤니티 기능

목표: 서버 구성원이 경기 전후로 함께 놀 수 있는 기능을 제공한다.

기능:
- 경기 승부예측 투표
- 서버 내 예측 랭킹
- 오늘의 직관 체크인
- 응원팀별 역할 자동 부여
- 홈런, 역전, 승리 등 상황별 응원 메시지

완료 기준:
- 투표와 랭킹은 서버 단위로 분리된다.
- 역할 자동 부여는 관리자 권한과 Discord 권한 오류를 안전하게 처리한다.
- 커뮤니티 메시지는 과도하게 시끄럽지 않도록 켜고 끌 수 있다.

## Phase 6: 고급 편의 기능

목표: 장기 사용자가 매일 또는 매주 반복해서 찾는 기능을 강화한다.

기능:
- `/하이라이트`: 경기 하이라이트 링크 모음
- 개인 DM 알림
- 주간 리포트: 응원팀 성적, MVP, 다음 일정
- 포스트시즌 모드: 와일드카드, 준플레이오프, 플레이오프, 한국시리즈 브래킷
- 데이터 캐싱과 장애 대응 강화

완료 기준:
- 사용자는 서버 알림과 개인 알림을 분리해서 설정할 수 있다.
- 주간 리포트는 자동 발송과 수동 조회를 모두 지원한다.
- 포스트시즌 일정과 시리즈 전적을 일반 시즌과 구분해서 표시한다.

## 공통 설계 과제

- 데이터 출처: KBO 공식 사이트, 기록 사이트, 뉴스 출처별 신뢰도와 갱신 주기를 정리한다.
- 캐싱: 실시간성이 필요한 데이터와 하루 단위 데이터의 캐시 정책을 분리한다.
- 저장소: 서버 설정, 알림 발송 이력, 예측 결과, 사용자 설정을 구분해서 저장한다.
- 오류 처리: 데이터 수집 실패, Discord API 실패, 권한 부족, 설정 누락을 사용자 친화적으로 응답한다.
- 테스트: 데이터 파싱, 명령어 응답 렌더링, 설정 저장, 알림 중복 방지를 우선 검증한다.
- 운영: 봇 시작 시 설정 검증, 로그, 헬스체크, Docker 실행 흐름을 유지한다.

## 추천 진행 순서

1. Phase 1A: `/오늘경기` 명령의 응답 형식과 데이터 흐름 정리
2. Phase 1B: `/순위` 명령 정리
3. Phase 1C: `/스코어` 명령 정리
4. Phase 1D: `/팀` 명령 추가 또는 정리
5. Phase 2A: 서버별 응원팀 설정
6. Phase 2B: 경기 시작/종료 알림
7. Phase 3 이후는 MVP 사용성이 확인된 뒤 진행

## 다음 작업 후보

가장 먼저 진행할 작업은 Phase 1A `/오늘경기`가 좋다. 이유는 KBO 봇의 핵심 사용 흐름이고, 이후 실시간 스코어와 알림 기능의 데이터 모델을 잡는 기준점이 되기 때문이다.
