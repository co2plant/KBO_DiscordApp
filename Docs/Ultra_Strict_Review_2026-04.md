# 초엄격 리뷰 보고서 (2026-04)

## 결론
- **운영 승인: 거절 (REJECTED)**
- 현재 상태는 "초기 안정화" 수준이며, **즉시 운영 투입 시 장애 확률이 높음**.

## Blocker (반드시 해결)

### B1. 호출되는 DB 함수가 실제로 없음 (즉시 런타임 에러)
- `kbo_crawler.py`는 `database.update_game_and_score(...)`를 호출함.
- `database.py`에는 해당 함수 정의가 없음.
- 결과: 스케줄 갱신 경로 진입 시 `AttributeError` 가능.

### B2. 스케줄러 파라미터 선언 불명확
- `@tasks.loop(hour=6)` 사용.
- 실행 의도가 매일 06:00인지, 6시간 주기인지 코드만 보고 확정 불가.
- 운영 코드에서는 해석 여지가 있는 선언을 금지해야 함.

### B3. Selenium 드라이버 정리 누락
- 여러 크롤링 함수에서 `driver.quit()`이 보장되지 않음.
- 장애/예외 시 프로세스 누수 가능.

### B4. 예외 처리 정책 부재
- `database.py`에 bare except 다수.
- 장애 원인 분류 불가, 자동 복구/알림 체계 연결 불가.

## High Risk (운영 전 강력 권고)

### H1. 건강상태 체크가 기능을 검증하지 않음
- compose healthcheck가 config.json 파싱만 확인.
- Discord 인증 실패/DB 단절/크롤러 오류를 감지하지 못함.

### H2. 의존성 버전 미고정
- Dockerfile에서 패키지 최신으로 직접 설치.
- 빌드 시점마다 동작 달라질 수 있음.

### H3. 비즈니스 로직 가정 오류 가능성
- `kbo.py` 일정 명령에서 월요일(`weekday() == 0`)은 무조건 경기 없다고 처리.
- 리그 일정 변동 시 오탐 가능.

## Medium

### M1. 네이밍/오탈자 품질 이슈
- `select_game_and_scord` 오탈자 함수명 유지.
- 유지보수 가독성 저하.

### M2. 테스트 커버리지 부족
- AST 형태 점검은 있으나 실제 DB/크롤링/Discord 상호작용 테스트는 없음.

## 통과 조건 (이 문서 기준)

아래를 모두 만족해야 "승인"으로 변경 가능:
1. `update_game_and_score` 구현 또는 호출 제거로 런타임 경로 정상화
2. 스케줄러 선언을 명시적 `time=`/`hours=`로 수정
3. 크롤러 함수 전부 `try/finally driver.quit()` 보장
4. bare except 제거 + 구조화 로깅
5. `/health` 명령 추가 및 compose healthcheck 연동
6. 의존성 고정(`requirements.txt` 또는 lock 기반)
7. 최소 통합 테스트(스테이징) 통과
