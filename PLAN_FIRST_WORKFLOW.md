# Plan-First Workflow

이 저장소는 문서 기반 개발을 기준으로 작업합니다. 기준 문서는 `docs/ROADMAP.md`입니다.

## 작업 순서

1. `docs/ROADMAP.md`에서 작업 후보와 현재 상태를 확인합니다.
2. 구현 전에 Spec을 작성해 목표, 범위, 제외 범위, 사용자 영향, 완료 기준을 확정합니다.
3. Spec 확정 후 Plan을 작성해 변경 방식, 영향 범위, 실패 모드, 테스트 방법, 커밋 단위를 확정합니다.
4. Spec과 Plan이 모두 확정된 뒤에만 코드를 변경합니다.
5. 단계가 끝날 때마다 검증 명령을 실행하고, 통과한 단위만 커밋합니다.
6. 머지와 배포는 사용자 요청이 있을 때만 진행합니다.

## 커밋 기준

- 문서, 구현, 테스트/검증은 가능한 한 분리해서 커밋합니다.
- 한 커밋에는 하나의 독립 작업만 담습니다.
- 커밋 전에는 해당 단계의 검증 명령이 통과해야 합니다.
- 사용자가 직접 커밋하므로, 작업자는 각 단계가 끝날 때 추천 커밋 메시지를 제공합니다.

## 문서 기준

- Canonical roadmap: `docs/ROADMAP.md`
- 사용자 실행 안내: `README.md`
- 오래된 `docs/superpowers/*` 문서는 참고용이며, 현재 상태 판단은 `docs/ROADMAP.md`를 우선합니다.

## 구현 금지 조건

- Spec이 없으면 새 기능 구현을 시작하지 않습니다.
- Plan이 없으면 구조 변경이나 데이터 변경을 시작하지 않습니다.
- 사용자 승인 없이 머지, 배포, destructive git 명령을 실행하지 않습니다.
