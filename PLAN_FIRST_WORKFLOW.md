# Plan-First Workflow

이 저장소의 다음 작업은 플랜을 먼저 확정한 뒤 브랜치에서 단계적으로 진행합니다.

## 작업 방식
1. 먼저 이번 작업의 목표, 범위, 제외 범위, 검증 조건을 문서로 적습니다.
2. 전용 작업 브랜치에서만 진행합니다.
3. 각 단계는 작은 단위로 나누고, 단계가 끝날 때만 커밋합니다.
4. 머지는 사용자 요청이 있기 전까지 하지 않습니다.
5. 추가 실험이 필요하면 현재 작업 브랜치에서 보조 브랜치를 더 만들어 분리합니다.

## 현재 기준 브랜치
- 작업 브랜치: `sisyphus/phase1-stability`

## 현재 실행 순서
1. Phase 1: standings 저장 경로 일관성 정리
2. Phase 2: Selenium 드라이버 종료 보장
3. Phase 3: 설정 검증과 런타임 시작 경계 정리
4. Phase 4: 검증 명령과 문서 정렬

## 커밋 규칙
- 한 단계가 끝나기 전에는 커밋하지 않습니다.
- 커밋 전에는 해당 단계의 테스트나 검증 명령이 먼저 통과해야 합니다.
- 각 커밋은 한 단계 또는 한 독립 작업만 담습니다.

## 현재 작업 기준
- 세부 단계별 QA와 통과 조건은 내부 플랜 문서 `.sisyphus/plans/kbo-phase1-stabilization.md`를 기준으로 진행합니다.
- 외부에 남는 기준점은 이 문서와 현재 작업 브랜치입니다.
